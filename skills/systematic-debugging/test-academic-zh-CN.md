# 学术测试：系统化调试技能 (Academic Test: Systematic Debugging Skill)

你可以访问 `skills/debugging/systematic-debugging` 这个技能。

请读取该技能，并且**只根据技能本身写的内容**回答以下问题：

1. 系统化调试的四个阶段分别是什么？
2. 在尝试任何修复之前，你必须做什么？
3. 在 Phase 3 中，如果你的第一个假设不起作用，应该怎么做？
4. 该技能如何看待“一次修多个东西”？
5. 如果你并没有完全理解这个问题，应该怎么办？
6. 对于简单 bug，是否存在可以跳过流程的情况？

在适用的地方，请附上技能中的直接引文。
