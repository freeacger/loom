# Changelog

## 2026-03-16

### 改动原因
从认知科学、脑科学、视觉传达三个维度分析交互体验：
- 工作记忆限制：多轮问答中用户需同时追踪已回答内容、当前问题、剩余轮数，Session Map 将"剩余轮数"外化，释放工作记忆槽位
- 杏仁核不确定性反应：不知道还有几轮会轻度激活压力反应，明确的会话地图提供确定性
- 预测编码：固定卡片格式让大脑准确预测结构，减少预测错误消耗的神经资源
- F型阅读模式：选项换行后每项独立成行，命中率显著高于行内排列
- 决策疲劳：开放式问题要求从头构建答案（系统2），选项列表降低为选择任务（系统1）
- 发散性不足：固定 2-3 个方案容易收敛到最显眼选项，4 个方案强制更广泛探索

### 改动项
1. **新增 Session Map 节**：开场声明预计问题数和议程主题，消除会话长度焦虑
2. **更新 Checklist 第 3 条**：方案数从 2-3 改为固定 4 个，确保发散性
3. **更新 Exploring approaches 节**：散文改为 4 行对比表格 + 推荐块（支持单选和组合推荐）
4. **更新 Understanding the idea 节**：追加 Question Card 格式引用
5. **新增 Question Card Format + Batching Rule 节**：固定问题卡片格式（背景/问题/选项换行），批量规则限定为纯事实类问题且不超过 3 张
6. **替换 Key Principles 节**：6 条扩展为 8 条，纳入新原则
